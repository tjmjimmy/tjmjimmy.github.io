(4)查

通过/select搜索索引，Solr制定一些参数完成不同需求的搜索：

(4.1)q

查询字符串，必须的.

(4.2)fq

fq是（filter query）过虑查询；作用：在q查询符合结果中同时是fq查询符合的。

  • 过滤查询价格从1到20的记录：id:[1 TO 20]
  • 使用“*”表示无限：id:[20 TO *]（表示20以上）；id:[* TO 20]（表示20以下）
(4.3)sort

排序功能，格式：field name + desc|asc[,field name + desc|asc]…。例如：

(4.4)start, rows
  • start - 分页显示使用，开始记录下标，从0开始
  • rows - 指定返回结果最多有多少条记录，配合start来实现分页。
(4.5)fl

指定返回那些域内容，用逗号或空格分隔多个。

(4.6)df

指定默认搜索域，如果在q搜索条件中没有指定搜索某个域，那么在这里就要指定搜索域

(4.7)wt

wt是writer type的缩写，指定输出格式，可以有 xml, json, php, phps。

(4.8)hl

表示是否高亮 ,设置高亮Field，设置格式前缀和后缀

(5)DataImport插件导入数据

  1. 将solr-6.6.0\dist路径下的solr-dataimporthandler-6.6.0.jar、solr-dataimporthandler-extras-6.6.0.jar复制到tomcat下solr工程的solr\WEB-INF\lib路径下（其实这个步骤已经在部署solr时已经执行了）
  2. 将相关数据库的jar包复制到tomcat下solr工程的solr\WEB-INF\lib路径下，我这里用的MySQL数据库，所以使用mysql-connector-java-5.1.7-bin.jar
  3. 配置solrhome下solrCore的solrconfig.mxl文件，添加一个requestHandler。你想导入哪个solrCore就配置相应solrCore的solrconfig.xml文件，例如我配置的是solrhome\collection2\conf\solrconfig.xml文件
